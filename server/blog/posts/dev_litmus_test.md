# Basic Developer Litmus Test
Here is an quick litmus test on programmer quality that I run into on the job, it normally happens early on in collaboration efforts.

If a developer does not have a strong understanding of git and the various services that leverage it, I make the assumption they are not a good programmer.

They either haven't taken the time or aren't capable of learning this CLI tech that's integral to almost every software project worked on in modern development. I'm guessing they are miss a lot of context in anything we're involved in now, too.
